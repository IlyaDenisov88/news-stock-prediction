Архитектура дипломной работы

1. Общая идея

Тема работы:
Прогнозирование краткосрочной динамики акций на основе рыночных данных и новостного фона.

Основная гипотеза:
Новостной фон содержит дополнительную информацию о будущей краткосрочной динамике акций, и добавление новостных признаков к рыночным данным может улучшить качество прогноза.

Основная постановка задачи:
- прогнозировать изменение цены на следующий интервал времени;
- в качестве основного таргета использовать log return;
- в качестве дополнительной постановки использовать направление движения up/down.

Почему именно так:
- log return лучше подходит для финансовых временных рядов, чем абсолютная цена;
- его удобнее сравнивать между разными акциями;
- бинарное направление движения можно получить дополнительно и использовать как вторую задачу.

2. Логика всей системы

Система строится как последовательный pipeline:
- загрузка рыночных данных;
- загрузка новостей;
- очистка и синхронизация данных по времени;
- построение признаков;
- формирование обучающего датасета;
- обучение моделей;
- прогноз следующего интервала;
- после наступления следующего интервала вычисление ошибки;
- дозагрузка новых новостей и котировок;
- повторение цикла.

То есть логика соответствует твоей исходной идее:
- собрали данные;
- обработали;
- предсказали движение на завтра или на следующий час;
- дождались фактического значения;
- посчитали ошибку;
- дозагрузили новые данные;
- повторили процесс.

3. Входные данные

3.1 Рыночные данные
Для каждой акции собираются:
- datetime;
- open;
- high;
- low;
- close;
- volume;
- value, если доступно;
- при необходимости данные по индексу MOEX как внешний рыночный ориентир.

Желательно, чтобы все данные были приведены к одному таймфрейму:
- либо дневному;
- либо часовому.

Если уже есть часовые данные, это хороший вариант для краткосрочного прогнозирования.

3.2 Новостные данные
Для каждой новости сохраняются:
- datetime публикации;
- заголовок;
- текст новости, если есть;
- источник;
- ссылка;
- тикер или компания, если новость уже привязана к активу;
- дополнительные метки, если получится извлечь.

Если пока есть только заголовки, этого уже достаточно для первой версии модели.

4. Предобработка данных

4.1 Предобработка рыночных данных
Нужно:
- отсортировать записи по времени;
- убрать дубликаты;
- проверить пропуски;
- привести типы признаков к числовому формату;
- нормализовать часовые пояса и формат времени.

Далее строятся рыночные признаки:
- simple return;
- log return;
- rolling mean;
- rolling std как прокси волатильности;
- high-low spread;
- close-open difference;
- volume change;
- lag features за несколько предыдущих интервалов.

4.2 Предобработка новостей
Нужно:
- удалить дубликаты;
- нормализовать дату и время публикации;
- привести текст к единому формату;
- при необходимости очистить HTML, спецсимволы и мусор;
- удалить явно нерелевантные публикации.

Если используются заголовки:
- сохранить headline как основной текстовый сигнал.

Если используется полный текст:
- можно отдельно хранить headline и body.

5. Синхронизация новостей и котировок

Это один из самых важных этапов.

На каждом моменте времени t модель должна видеть только ту информацию, которая была доступна до t.
Нельзя включать новости, вышедшие после момента прогноза.

Практическая схема:
- выбирается таймфрейм, например 1 час;
- для интервала t собираются все новости, опубликованные до конца этого интервала;
- на основе этих новостей строятся агрегированные признаки;
- target относится к следующему интервалу t+1.

Пример:
- новости с 10:00 до 10:59 используются вместе с рыночными признаками на момент 10:59;
- модель прогнозирует изменение цены на интервал 11:00-11:59.

Если использовать дневной таймфрейм, логика такая же:
- новости до конца торгового дня t используются для прогноза дня t+1.

6. Формирование target

6.1 Основной target
Рекомендуемый основной target:
log_return(t+1) = ln(close(t+1) / close(t))

Плюсы:
- устойчивее, чем raw price;
- корректнее для финансовых рядов;
- удобен для сравнения между бумагами.

6.2 Дополнительный target
Бинарное направление движения:
- 1, если log_return(t+1) > 0;
- 0, если log_return(t+1) <= 0.

Это позволит сделать вторую постановку задачи как классификацию.

7. Построение признаков

7.1 Рыночные признаки
Минимальный набор:
- open, high, low, close, volume;
- return;
- log return;
- rolling mean за несколько окон;
- rolling volatility;
- lagged close и lagged return.

7.2 Новостные признаки первого уровня
Это простой и интерпретируемый вариант:
- количество новостей за интервал;
- средний sentiment;
- максимальный и минимальный sentiment;
- стандартное отклонение sentiment;
- доля позитивных и негативных новостей.

7.3 Новостные признаки второго уровня
Это более сильный вариант:
- эмбеддинги новостей из BERT-подобной модели;
- усреднение эмбеддингов за интервал;
- attention pooling или weighted aggregation;
- отдельно можно добавить embedding заголовка и embedding полного текста.

7.4 Дополнительные признаки
Если будет время, можно добавить:
- день недели;
- время суток;
- расстояние до открытия/закрытия торгов;
- признак выхода важной корпоративной новости;
- рыночный фон по индексу.

8. Формирование датасета для модели

После построения признаков нужно сформировать обучающие примеры.

Есть два варианта.

8.1 Табличный вариант
Каждый объект соответствует одному моменту времени t.
Признаки:
- агрегированные рыночные признаки на t;
- агрегированные новостные признаки на t.
Target:
- log_return(t+1) или direction(t+1).

Это подходит для:
- Linear models;
- Random Forest;
- XGBoost;
- CatBoost;
- MLP.

8.2 Последовательный вариант
Каждый объект - это окно длины seq_len.
Признаки:
- последовательность рыночных и новостных признаков за интервалы t-seq_len+1 ... t.
Target:
- значение на t+1.

Это подходит для:
- LSTM;
- GRU;
- temporal attention models.

Практически для диплома удобно иметь оба варианта:
- сначала табличный baseline;
- потом последовательную нейросетевую модель.

9. Набор моделей для диплома

9.1 Модель A: Market-only baseline
Вход:
- только рыночные признаки.

Возможные модели:
- Linear Regression / Logistic Regression;
- XGBoost;
- LSTM.

Назначение:
- понять, что можно предсказать без новостей.

9.2 Модель B: Market + sentiment
Вход:
- рыночные признаки;
- агрегированные sentiment-признаки по новостям.

Возможные модели:
- XGBoost;
- MLP;
- LSTM.

Назначение:
- проверить, дает ли простой текстовый сигнал прирост качества.

9.3 Модель C: Market + news embeddings
Вход:
- рыночные признаки;
- агрегированные эмбеддинги новостей.

Возможные модели:
- MLP на табличных агрегатах;
- late fusion architecture;
- LSTM/GRU с объединением признаков на каждом временном шаге.

Назначение:
- получить более сильную модель с богатыми текстовыми представлениями.

9.4 Модель D: Advanced extension
Возможные расширения:
- hierarchical attention;
- event-driven features;
- graph-based approach;
- LLM-based factor extraction.

Назначение:
- не как обязательное ядро, а как возможное развитие исследования.

10. Рекомендуемый экспериментальный дизайн

Чтобы диплом выглядел убедительно, логика экспериментов должна быть ступенчатой.

Этап 1.
Сравнить market-only baseline и market+sentiment.

Этап 2.
Сравнить market+sentiment и market+embeddings.

Этап 3.
Если получится, сравнить разные способы агрегации новостей:
- mean pooling;
- attention pooling;
- title only против title+body.

Основная идея сравнений:
- помогают ли новости вообще;
- помогает ли более богатое текстовое представление по сравнению с простым sentiment.

11. Обучение и валидация

Нельзя использовать случайное перемешивание данных.

Нужна time-series validation:
- train на раннем периоде;
- validation на следующем отрезке;
- test на самом позднем периоде.

Возможные схемы:
- fixed split;
- rolling window;
- expanding window.

Для диплома достаточно:
- один понятный train/valid/test split;
- при желании дополнительно rolling evaluation.

12. Метрики качества

Для регрессии:
- MAE;
- RMSE;
- MAPE с осторожностью;
- directional accuracy как вспомогательная метрика.

Для классификации направления:
- Accuracy;
- Precision;
- Recall;
- F1-score;
- ROC-AUC, если будет вероятностный выход.

Для финансовой интерпретации можно дополнительно смотреть:
- hit ratio по направлению;
- средний результат простой торговой стратегии без сложного бэктеста.

Но в дипломе важно не перегружать. Базовых метрик достаточно.

13. Ежедневный или пошаговый рабочий цикл модели

Эта часть хорошо соответствует твоей интуитивной схеме.

Шаг 1.
Загружаются новые котировки и новые новости.

Шаг 2.
Новые данные очищаются и приводятся к общему формату.

Шаг 3.
Новости агрегируются в признаки за новый интервал.

Шаг 4.
Формируется новый объект для прогноза.

Шаг 5.
Модель выдает прогноз для следующего интервала:
- expected log return;
- или вероятность роста.

Шаг 6.
После наступления фактического следующего интервала вычисляется ошибка прогноза.

Шаг 7.
Новый факт добавляется в датасет.

Шаг 8.
По выбранному расписанию выполняется дообучение или переобучение модели.

То есть operational cycle выглядит так:
- ingest new data -> preprocess -> feature build -> predict -> observe fact -> evaluate error -> update dataset -> retrain.

14. Как лучше организовать обновление модели

Есть три возможные схемы.

Схема 1. Пакетное переобучение раз в период
- например раз в неделю или раз в месяц;
- самый простой и надежный вариант.

Схема 2. Скользящее окно обучения
- модель всегда обучается на последних N интервалах;
- позволяет адаптироваться к меняющемуся рынку.

Схема 3. Почти онлайн-обновление
- после каждого нового интервала обновляется модель или датасет.

Для диплома оптимально:
- собирать новые данные регулярно;
- переобучать модель пакетно по расписанию;
- в тексте указать, что онлайн-обновление рассматривается как дальнейшее развитие.

15. Предлагаемая структура практической части

Глава 1. Подготовка данных
- описание источников котировок и новостей;
- формат хранения;
- очистка и синхронизация.

Глава 2. Построение признаков
- рыночные признаки;
- sentiment features;
- text embeddings;
- формирование target.

Глава 3. Модели
- baseline без новостей;
- модель с sentiment;
- модель с embeddings.

Глава 4. Эксперименты
- схема разбиения по времени;
- метрики;
- сравнение моделей.

Глава 5. Анализ результатов
- помогают ли новости;
- какая группа признаков полезнее;
- ограничения подхода.

16. Наиболее разумная итоговая архитектура для диплома

Если выбирать один основной маршрут, то он выглядит так:

Данные:
- котировки по акциям;
- новости по этим акциям.

Предобработка:
- очистка;
- нормализация дат;
- синхронизация по интервалам времени.

Признаки:
- market features;
- sentiment features;
- news embeddings.

Targets:
- основной: next-period log return;
- дополнительный: next-period direction.

Модели:
- baseline market-only;
- market + sentiment;
- market + embeddings.

Оценка:
- time-series split;
- regression metrics;
- classification metrics для дополнительной задачи.

Обновление:
- регулярная дозагрузка новостей и котировок;
- пересчет признаков;
- периодическое переобучение модели.

17. Краткий вывод

Для дипломной работы наиболее логично строить архитектуру как многоэтапный pipeline, в котором рыночные данные и новостной поток сначала приводятся к общему временному представлению, затем преобразуются в признаки, после чего используются для прогнозирования краткосрочного изменения цены акции. Базовая экспериментальная схема должна включать три уровня моделей: без новостей, с простыми новостными признаками и с глубокими текстовыми представлениями. Такая архитектура одновременно научно обоснована, воспроизводима и реалистична по объему для дипломного проекта.
