# news-stock-prediction

Дипломный проект по анализу влияния новостного фона на краткосрочную динамику цен акций российского фондового рынка с применением методов машинного обучения.

## Идея проекта

Цель работы - проверить, улучшают ли новости качество прогноза по сравнению с моделями, которые используют только рыночные данные.

Проект строится вокруг задачи краткосрочного прогнозирования:
- процентного изменения цены акции;
- направления движения up/down.

Основной акцент делается на объединении двух источников информации:
- рыночных данных Московской биржи;
- новостного фона из финансовых и деловых источников.

## Основные этапы

- сбор котировок MOEX;
- сбор новостей;
- предобработка и синхронизация данных по времени;
- построение рыночных и текстовых признаков;
- обучение baseline-модели только по рыночным данным;
- обучение моделей с признаками, извлеченными из новостей;
- анализ результатов.

## Архитектура в сжатом виде

Общий pipeline работы:

1. Загружаются исторические котировки акций.
2. Загружаются новости с выбранных источников.
3. Данные очищаются, нормализуются и приводятся к общей временной шкале.
4. Для каждого временного интервала агрегируются рыночные и новостные признаки.
5. Формируется `target` для следующего интервала.
6. Обучаются несколько моделей:
   - `market-only baseline`;
   - `market + sentiment`;
   - `market + news embeddings`.
7. Модели сравниваются на временном разбиении `train/validation/test`.
8. После поступления новых данных цикл может повторяться:
   - дозагрузка новостей и котировок;
   - пересчет признаков;
   - прогноз;
   - расчет ошибки;
   - переобучение.

Постановка задачи:
- основной `target`: `log_return(t+1) = ln(close(t+1) / close(t))` (логарифмическая доходность);
- дополнительный `target`: направление движения цены на следующий интервал.

## Источники данных

### Котировки

Сбор котировок выполняется через официальный API Московской биржи `MOEX ISS`.

Используются исторические данные по торгам, включая:
- `open`;
- `high`;
- `low`;
- `close`;
- `volume`;
- при наличии дополнительные торговые показатели.

Для исследования выбраны наиболее ликвидные акции российского фондового рынка - топ-50 компаний по обороту. Такой выбор нужен, чтобы:
- снизить уровень случайного шума;
- получить более устойчивую реакцию рынка на новости;
- сделать выборку более репрезентативной.

Основной рабочий период данных:
- с `2023` по `2026` год.

Для быстрого тестирования отдельных этапов также были использованы более короткие интервалы.

Основной таймфрейм котировок:
- часовой.

Это компромисс между:
- чувствительностью к краткосрочной реакции рынка на новости;
- уровнем шума;
- вычислительной сложностью.

### Новости

Сбор новостей осуществляется парсингом HTML-страниц финансовых и деловых источников:
- InvestFunds;
- Interfax;
- Kommersant;
- Smart-Lab.

Для каждой новости извлекаются:
- источник;
- дата и время публикации;
- заголовок;
- ссылка на материал.

Заголовки позволяют получить компактное и информативное представление новости без усложнения пайплайна обработкой полных текстов. Новостные данные собираются за тот же период, что и котировки, чтобы корректно сопоставлять информационный фон и рыночную динамику.

## Формирование признаков

В проекте рассматриваются несколько уровней признаков.

### Рыночные признаки

- цены и объемы;
- процентные изменения;
- `log return` - логарифмическое процентное изменение;
- лаговые признаки;
- скользящие средние;
- оценки волатильности.

### Новостные признаки

Базовый уровень:
- количество новостей за интервал;
- агрегированный sentiment;
- статистики sentiment по интервалу.

Продвинутый уровень:
- эмбеддинги новостей;
- агрегирование эмбеддингов по временному окну;
- объединение текстовых и рыночных признаков в одной модели.

## Экспериментальный дизайн

Модели строятся по ступеням:

1. `Market-only baseline`
2. `Market + sentiment`
3. `Market + news embeddings`

Такая схема позволяет ответить на два ключевых вопроса:
- помогают ли новости вообще;
- дают ли сложные текстовые представления преимущество над простыми sentiment-признаками.

```text
src/
  data_collection/

    news_parse.py
  data_processing/

  modeling/

notebooks/
  demo_SBER_data.ipynb

```

Ключевые директории:
- `src/data_collection` - сбор котировок и новостей;
- `src/data_processing` - объединение и подготовка данных;
- `src/modeling` - baseline-модели;
- `notebooks` - исследовательские ноутбуки и демонстрации;
- `articles` - краткие саммари научных статей и рабочие материалы по архитектуре диплома.

## Запуск

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Дальнейший запуск зависит от конкретного этапа пайплайна:
- сбор котировок;
- сбор новостей;
- merge и подготовка датасета;
- обучение моделей.

## Текущий статус

На текущем этапе:
- собран пайплайн загрузки рыночных данных;
- реализован сбор новостей;
- определен основной временной горизонт исследования;
- сформулирована архитектура дипломной работы;
- подбираются и сравниваются варианты моделирования.

## План экспериментов

Ближайший рабочий порядок:

1. Проверить качество собранных новостных данных.
2. Сопоставить новости и котировки по времени.
3. Сформировать единый датасет с признаками и `target`.
4. Обучить baseline-модель только на рыночных данных.
5. Добавить sentiment-признаки.
6. Добавить текстовые эмбеддинги.
7. Сравнить модели на временном разбиении.
